# 语料预审库

> 存放待审核的语料标注文件。文件从 App 中栏编辑器导出后先进入此目录，审核完成后移入 `../corpus/`。

## 目录结构

```
corpus-review/
├── README.md
└── 美食酒水/
    ├── 休闲美食/
    ├── 坚果蜜饯/
    ├── 冲饮茗茶/
    ├── 滋补保健/
    ├── 酒水饮料/
    ├── 粮油调味/
    ├── 水果蔬菜/
    └── 肉蛋海鲜/
```

品类路径与 App 标准三级类目一致（`level1/level2/level3`）。

## 文件规范

- 格式：仅 `.json`（v2 schema：`corpus-图文绑定-v2`）
- 命名：`{商品名缩写}_{品类}.json`，如 `认养一头牛酸奶_乳制品.json`
- 编码：UTF-8（无 BOM）

## 审核流程

1. 用户完成编辑后，从 App 导出或发布时自动生成 JSON 到预审库对应品类目录
2. 运营审核文件内容（图文绑定、图片分类、内容摘要）
3. 确认无误后将 JSON 移入 `../corpus/` 对应目录
4. 移入后自动加入 RAG 索引

## 审核要点

- [ ] 品类路径是否正确（与 App 一致）
- [ ] 图片 ID 是否连续、无遗漏
- [ ] 每个模块的 `segments` 是否完整覆盖文案
- [ ] `imageContentSummary` 是否准确描述了图片中的文字/数字信息
- [ ] `styleTag` 是否与原文案风格一致
