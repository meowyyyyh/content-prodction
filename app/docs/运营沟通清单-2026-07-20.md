## 二、定评分

### 2.1 语料评分标准

| 决策点 | 选项 | 建议 |
|------|------|------|
| 打分方式 | A. 导出时弹窗打 1-5 星 / B. 事后统一评 / C. 系统自动算 | **A+C**：系统默认按编辑率给初始分，运营可在语料库手动调整 |
| 评分维度 | 只看文案质量？还是图片+转化？ | **先只看文案质量**，简单起步 |
| 分数范围 | 1-5 星 / 1-100 分 | **1-5 星** |

## 四、中栏编辑反馈

| 问题 | 为什么问 |
|------|------|
| 采纳 AI 文案后一般改多少？（大改/微调/不改） | 验证编辑率基线 |
| 最常改什么？（字数/emoji/措辞/图片位置？） | 指导学习优先级 |
| 快捷按钮够用吗？想加什么？ | 扩展 chip 候选 |
| 有没有反复改同一处的经历？哪个模块？ | 定位 prompt 盲区 |
| 图片拖拽好用吗？图文排版改得多吗？ | 图片体验反馈 |
| 现在三版里最常采纳哪个版本？ | 验证冷启动默认风格 |
| 有没有"想生成但做不到"的场景？ | 发现功能缺口 |

## 五、冷启动预期

告诉运营：

- **前几篇**：和现在差不多，系统还在学
- **用一周（~15-20 篇）后**：顺序越来越对，改得越来越少
- **核心逻辑**：导出越多 = AI 学得越好 = 用得越爽
