# 快稿种草小助手 — 项目进展汇报

> 2026-07-20 · 向领导汇报用

---

## 一、项目概述

快稿种草小助手是一个 AI 驱动的商品笔记内容生成工具。运营输入商品信息 + 上传图片，系统自动生成三版图文混排的种草笔记，支持逐模块采纳编辑、一键导出。核心定位：从"AI 帮你写"到"AI 越来越像你写"——系统通过学习运营的采纳、编辑和导出行为，逐步适配个人风格。

**发布方式**：独立网址（已有自有服务器，可直接部署）。

## 二、当前整体进度

### 2.1 已完成 ✅

| 模块 | 内容 |
|------|------|
| 核心生成引擎 | AI 文案三版并行生成（V1 默认风格 + V2/V3 动态风格），流式输出 |
| 图片处理 | doubao vision API 全维度图片分析（类型识别/OCR/内容摘要/模块建议） |
| 类目体系 | 15 个一级类目 + 三级查找链（三级精确→二级覆盖→一级默认→兜底），美食酒水 14 个模块 |
| 语料基础设施 | v2.1 schema 图文绑定语料结构、导出自动存入预审库（corpus-review/）、图片指纹去重 |
| 编辑功能 | contentEditable 中栏编辑、模块拖拽排序、快捷按钮（扩充/口语化/排版等）、对话框自由改写 |
| 文案风格 | 6 种纯风格体系（xiaohongshu/girlfriend/fun/minimalist/premium/senior） |
| 正式语料 | **1 篇**（认养一头牛每日吨吨木姜子香茅酸奶，xiaohongshu 风格） |

| # | 决议 | 影响 |

## 四、当前阻点

| # | 阻点 | 影响 | 预计缓解 |
|:--|------|------|:--:|
| 1 | **种子语料严重不足** | 正式语料仅 1 篇（xiaohongshu 风格），语料搜索 API 上线后无数据可用，V2/V3 只能兜底默认顺序。其他 5 种风格完全没有语料 | 运营已拿到评分表，预计 **1 周**内交付美食酒水类目 30-50 篇 |

| 3 | **Elo 排名系统未开发** | V2/V3 的风格选择目前是写死的（xiaohongshu/fun），无法根据运营偏好动态调整 | 三引擎重构完成后启动 |

```
├── AI 提炼卖点方案设计 → 开发
└── 多规格表单方案设计 → 开发

第 2 层（依赖语料入库）
├── Step 4：语料库搜索 API + 动态评分
├── Step 2+5：三版独立排序 + 语料参考接入
├── Step 3：采纳快照 + 导出学习 + buildCorpusJSON 扩展

└── 一键发布方案讨论 + 技术对接
```

| 时间 | 里程碑 |
|------|------|
| 第 1 周 | 运营交付种子语料 30-50 篇入库；同步完成 Step 1（左面板去排序化）、物流售后 prompt、Elo 基础设施、AI 提炼卖点方案设计 |
| 第 2 周 | Step 4（语料搜索 API） + Step 2+5（三版独立排序）、AI 提炼卖点开发、多规格表单方案 → 开发 |
| 第 3 周 | Step 3（采纳快照 + 导出学习） + Step 6（用户画像类目隔离）、联调测试、上线使用 |

**第 3 周末**：运营可用版本上线。V1 开始学习用户行为，V2/V3 从种子语料获取顺序参考。Elo 冷启动期（2-4 周）。

> 目标：Elo 开始生效，V2/V3 风格动态调整，V1 越用越像运营。

| 时间 | 里程碑 |
|------|------|
| 第 4-6 周 | Elo 积累足够信号 → V2/V3 风格动态排序生效；V1 模块顺序学习生效（≥10 次导出后三级类目独立）；编辑率逐步下降 |
