# 种子语料入库评分体系 — 设计方案 v2

> 2026-07-20 · v2 合入 Claude Code 审阅意见 · 用于运营筛选历史优质笔记入库的评分标准
> 适用范围：美食酒水一级类目 · 冷启动期种子语料收集

---

## 一、核心认知：为什么要分级评分

三件事不能混在一起：

| 阶段 | 做什么 | 用什么标准 |
|---|---|---|
| **入库筛选**（现在要做的） | 运营从历史笔记里挑出"值得让 AI 学"的 | **人工评分卡**（本文档） |
| **入库后排序**（未来，已设计） | 池子里几十篇语料，优先引用谁 | **动态评分**（编辑率 × 引用 × 时效） |
| **运营日常产出**（未来，飞轮 L3） | 导出/发布自动进预审库 | **自动过滤**（编辑率 ≤ 50% 入库） |

现在做的是第一件事，运用于运营翻历史笔记的场景。

---

## 二、入库评分卡（5 维度，运营手动打分）

### 维度 1：模块完整度（权重 30%）

**考察什么**：笔记里有没有覆盖该品类应有的核心模块。

| 分数 | 标准 |
|:--:|------|
| 5 | 覆盖该品类全部核心模块 + 至少 1 个可选模块，结构清晰可辨 |
| 4 | 覆盖了全部核心模块 |
| 3 | 缺 1 个核心模块 |
| 2 | 缺 2-3 个核心模块 |
| 1 | 只有钩子和 CTA，中间全是自由文案，看不出模块边界 |

**美食酒水各组的核心模块参照**：

| 组别 | 适用二级类目 | 核心模块 |
|---|---|---|
| A 组 | 酒水饮料、休闲美食、坚果蜜饯、冲饮茗茶 | hook、taste、ingredient、price、trust、cta |
| A' 组 | 粮油调味（无配料表） | hook、taste、price、trust、cta（ingredient 不算核心） |
| B 组 | 水果蔬菜、肉蛋海鲜 | hook、taste、origin、price、trust、cta |
| C 组 | 滋补保健 | hook、taste、origin、price、trust、cta |

> 核心模块清单随类目组变化，运营不需要背——在任务单里给每个二级类目列出它的核心模块。

### 维度 2：图文绑定质量（权重 25%）

**考察什么**：图片有没有插在对应的文案段落里，还是全堆在末尾。

| 分数 | 标准 |
|:--:|------|
| 5 | 几乎每段文案都有对应配图，图片内容与相邻文案强相关（如成分段配配料表图） |
| 4 | 大部分文案段有对应配图，偶尔一两段无图 |
| 3 | 图文大体对应但有多处错位（如图在文前很远处）或大量图片堆在末尾 |
| 2 | 图片基本集中在笔记底部，中间只有文字 |
| 1 | 纯文字无图，或图片完全与文案无关 |

### 维度 3：信息密度（权重 25%）

**考察什么**：笔记里有没有硬信息（规格、价格、成分、产地），还是全是情绪文案。硬信息密度是 AI 生成电商文案最稀缺的资产，权重最高。

| 分数 | 标准 |
|:--:|------|
| 5 | 价格、规格、成分/配料、产地/奶源、保质期、适用人群等硬信息齐全，且数字精确 |
| 4 | 有价格 + 规格 + 成分/配料，其余部分有 |
| 3 | 有价格 + 至少 2 项硬信息 |
| 2 | 有价格但其他硬信息很少 |
| 1 | 全是感叹号和情绪词，没有可提取的硬信息 |

### 维度 4：风格特征（权重 15%）

**考察什么**：笔记有没有一个可辨识的风格范式，这对 AI 学习风格区分很关键。

| 分数 | 标准 |
|:--:|------|
| 5 | 风格鲜明且全程一致（如整篇都是"闺蜜种草"口吻，emoji 密度、句式结构统一），同时有至少 2 个明显可提取的风格特征（分段方式、语气词习惯、排版符号等）。**语言自然流畅，无模板感或拼接痕迹** |
| 4 | 风格清晰可辨，全程基本一致，语言自然 |
| 3 | 有风格倾向但中途偶尔走样，有少量生硬段落 |
| 2 | 风格模糊，看不出是什么风格 |
| 1 | 风格混乱、生硬拼接、或与品类完全不匹配 |

**可辨识的风格标签**（运营打标用，非评分用）：xiaohongshu 种草 / 专业测评 / 闺蜜安利 / 高端介绍 / 趣味整活 / 干货清单

> **否决项**：如果笔记明显是 AI 生成或模板拼凑（整篇读起来僵硬、套路感重），直接不入库，不管其他维度得分多高。

### 维度 5：实战验证（权重 5%）

**考察什么**：这篇笔记在真实销售场景中的表现如何。注意：成交数据受商品热度、价格、促销力度影响很大，不能直接当笔记质量的代理变量——此维度仅作微弱加分。

**计分规则**（0-5 分，三项累加。无成交数据时此维度标记为 N/A，不计入总分，总分按剩余 4 个维度重新加权）：

| 子项 | 条件 | 加分 |
|---|---|---|
| 成交表现 | 成交订单量（近 30 天）≥ 同类目同价格带中位数 | +3 分 |
|  | 成交订单量（近 30 天）< 中位数但 > 0 | +1 分 |
|  | 无成交数据 | N/A（此维度不计入总分） |
| 可信度 | 退款率 ≤ 5% | +1 分 |
| 复用度 | 被用于 ≥ 3 个不同团/批次 | +1 分 |

**满分示例**：成交高于中位数(+3) + 退款率 3%(+1) + 被 5 个团用过(+1) = 5 分。

> **关于退款率的假设**：前提是做的是选品把关，商品本身没问题。因此高退款率更可能来自笔记过度承诺（夸大功效/描述不符），退款率低说明笔记诚实地呈现了商品。这个假设如果未来被推翻，需重新校准。

## 三、入库判定

**加权总分 ≥ 3.5 分 → 入库（建议）**

权重计算示例（某篇乳制品笔记，有成交数据）：

| 维度 | 原始分 | 权重 | 加权 |
|---|---|---|---|
| 模块完整度 | 4 | 30% | 1.20 |
| 图文绑定 | 3 | 25% | 0.75 |
| 信息密度 | 5 | 25% | 1.25 |
| 风格特征 | 4 | 15% | 0.60 |
| 实战验证 | 5 | 5% | 0.25 |
| **总分** | | | **4.05** ✅ |

**无成交数据的笔记计算示例**（实战验证 = N/A，剩余 4 维权重等比放大）：

| 维度 | 原始分 | 有效权重 | 加权 |
|---|---|---|---|
| 模块完整度 | 4 | 30/95 ≈ 31.6% | 1.26 |
| 图文绑定 | 3 | 25/95 ≈ 26.3% | 0.79 |
| 信息密度 | 5 | 25/95 ≈ 26.3% | 1.32 |
| 风格特征 | 4 | 15/95 ≈ 15.8% | 0.63 |
| 实战验证 | N/A | — | — |
| **总分** | | | **4.00** ✅ |

**降级兜底**：如果某个三级类目翻遍历史笔记都找不到 ≥3.5 分的，允许降级到 ≥3.0 分入库 1 篇"占位"，后续产出更高质量的再替换。

## 四、美食酒水 8 个二级类目的三级类目分配

**基础策略**：每个二级类目挑 3-5 个三级类目，每个三级类目至少 1 篇。

**增强策略**：成交量 TOP 3 的三级类目各拿 3-5 篇（不同风格标签），以支撑风格学习。其他三级类目保持 1 篇。

| 二级类目 | 推荐三级类目（按优先级） | 已有 | TOP 3 策略 | 核心模块 |
|---|---|---|---|---|
| **酒水饮料** | 乳制品 ✓、白酒、果汁饮料、茶饮、(5选1) | 1 | 乳制品已有 1 篇，再补 2 篇不同风格；白酒/果汁各 1 | hook/taste/ingredient/price/trust/cta |
| **休闲美食** | 饼干冻干、方便速食、肉类零食、膨化食品、糕点面包 | 0 | TOP 3 三级各 2-3 篇（含不同风格），其余各 1 | hook/taste/ingredient/price/trust/cta |
| **坚果蜜饯** | 坚果炒货、蜜饯果铺、开心果/碧根果 | 0 | TOP 3 三级各 2-3 篇，其余各 1 | hook/taste/ingredient/price/trust/cta |
| **冲饮茗茶** | 成人奶粉、咖啡/奶茶、红茶/花茶/绿茶、蜂蜜 | 0 | TOP 3 三级各 2-3 篇，其余各 1 | hook/taste/ingredient/price/trust/cta |
| **滋补保健** | 燕窝、人参、养生茶/阿胶 | 0 | 三级类目共 3 个，各 1-2 篇覆盖不同风格 | hook/taste/origin/price/trust/cta |
| **粮油调味** | 大米、食用油、调味酱菜、(1选) | 0 | TOP 3 三级各 2-3 篇，其余各 1 | hook/taste/price/trust/cta |
| **水果蔬菜** | 时令水果、热带水果、地标特色、菌菇 | 0 | TOP 3 三级各 2-3 篇，其余各 1 | hook/taste/origin/price/trust/cta |
| **肉蛋海鲜** | 精选肉类、蛋类、鱼虾贝蟹、海产干货 | 0 | TOP 3 三级各 2-3 篇，其余各 1 | hook/taste/origin/price/trust/cta |

**预估总量**：约 30-50 篇（较 v1 的 23-39 有所增加，但核心类目的风格覆盖率大幅提升）。

**执行顺序建议**：酒水饮料已有 1 篇，优先级可放低——先把其他 7 个二级类目各凑齐至少 3 篇，再回来补酒水饮料。

> TOP 3 三级类目的具体清单由运营根据实际成交数据确定，Excel 任务单中提供"成交排名"辅助字段。

## 五、运营任务单模板

给运营发 Excel，每行一篇笔记。以下为列定义（附 Excel 公式）：

| 列 | 字段名 | 说明 |
|:--:|------|------|
| A | 商品名称 | 如"认养一头牛每日吨吨木姜子香茅酸奶" |
| B | 一级类目 | 固定"美食酒水" |
| C | 二级类目 | 下拉选择（8 个选项） |
| D | 三级类目 | 下拉选择（随二级类目联动） |
| E | 成交订单量(近30天) | 数字，无数据留空 |
| F | 成交排名 | 同二级类目下成交排序（用于确定 TOP 3） |
| G | 退款订单量(近30天) | 数字，无数据留空（填 0 或不填均可） |
| H | 退款率 | **自动计算**，公式：`=IF(F2>0,G2/F2,"")`。百分比格式 |
| I | 复用团数 | 数字，无数据填 0 |
| J | 风格标签 | 下拉：xiaohongshu/专业测评/闺蜜安利/高端/趣味/干货 |
| K | 模块完整度 (0-5) | 运营打分 |
| L | 图文绑定 (0-5) | 运营打分 |
| M | 信息密度 (0-5) | 运营打分 |
| N | 风格特征 (0-5) | 运营打分 |
| O | 实战验证 (0-5) | 自动计算，公式：`=IF(F2="","N/A",IF(F2>=50,3,IF(F2>0,1,0))+IF(H2<=0.05,1,0)+IF(I2>=3,1,0))` |
| P | 总分 | 自动计算，公式：`=IF(O2="N/A",K2*0.3/0.95+L2*0.25/0.95+M2*0.25/0.95+N2*0.15/0.95,K2*0.3+L2*0.25+M2*0.25+N2*0.15+O2*0.05)` |
| Q | 是否入库 | 自动判定，公式：`=IF(P2>=3.5,"✅ 入库",IF(P2>=3,"⚠ 可占位","❌"))` |
| R | 笔记原文链接 | URL 或内部文档 ID |
| S | Word 文档命名 | 自动拼接，公式：`=D2&"_"&E2&"_"&B2&".docx"` |
| T | 备注 | 运营可记录特殊情况 |

**使用说明**：
- 过渡期成交阈值（≥50 单 = +3 分，≥10 单 = +1 分）写死在公式里。等同类目中位数数据充足后切换为 `F2>=MEDIAN(...)` 表达式。
- 退款率由 H 列自动计算（退款订单量/成交订单量），运营只需填 F 和 G 列原始数字。
- 无成交数据时 O 列显示"N/A"，P 列自动按 4 维度重加权。
- S 列自动生成 Word 命名，运营保存 Word 时直接复制该列值。

**输出物**：Word 文档（保留原文案+配图+结构），命名规范 `{二级类目}_{三级类目}_{商品名}.docx`

## 六、与现有评分体系的关系

| 评分体系 | 谁打分 | 什么时候用 | 做什么决策 |
|---|---|---|---|
| **本方案（入库评分卡）** | 运营人工 | 笔记进入语料库之前 | 这篇笔记要不要进语料库 |
| **动态评分系统**（语料动态评分系统设计方案.md） | 系统自动 | 语料入库后，每次搜索时实时计算 | 池子里几十篇语料优先参考谁 |
| **数据飞轮 L3 过滤**（数据飞轮建设方案.md） | 系统自动 | 运营导出/发布时 | 编辑率 ≤ 50% 自动入库 |

### 6.1 入库评分 → 动态评分的衔接

入库评分是一次性的——用于决定笔记能否入库。入库后，动态评分系统（编辑率 × 引用 × 时效）从零开始计算。

**建议衔接机制**：入库总分作为动态评分的初始"种子值"——例如将入库总分映射为动态评分的初始引用分（`referenceCount = floor(入库总分 × 3)`），让精心筛选的笔记在冷启动期更容易被优先引用，而不是和自动入库的笔记站在同一起跑线。此机制需配合《语料动态评分系统设计方案.md》联动改造。

### 6.2 L3 飞轮自动入库的区分

L3 飞轮"编辑率 ≤ 50% 自动入库"会绕过人工评分。建议自动入库的笔记打 `source: "auto"` 标签，人工筛选的打 `source: "curated"`，以便后续做质量对比分析。

## 七、风险与开放问题

1. **成交中位数怎么定**：目前没有同类目同价格带的历史数据基线。第一批打分时过渡期用绝对阈值（≥50 单 = +3 分，≥10 单 = +1 分）。等攒够数据（每个二级类目 ≥10 条成交记录）后切换为中位数比较。

2. **退款率 5% 阈值的合理性**：美食酒水类目的行业退款率数据暂缺。建议先用第一批数据算出实际分布再调整阈值。

3. **风格打标的主观性**：同一个笔记，运营 A 觉得是"闺蜜风"，运营 B 觉得是"种草风"，这很正常。AI 不需要精确标签——它需要的是多样化的风格样本。打分保持大致一致即可，标签不要求 100% 准确。

4. **AI 生成/模板拼凑笔记**：设为否决项——明显 AI 生成或模板拼凑的笔记直接不入库。如果未来 AI 生成比例上升、否决过于频繁，再评估是否改为降分而非否决。

5. **无成交数据的笔记不会被系统性低估**：N/A 机制确保新笔记在实战验证维度不自扣分。但运营仍需手动判断新笔记的其他四个维度。
