# 智能类目识别 + 模块深推 执行方案 v1

> 2026-07-21 · 合并小co审阅反馈

---

## 改动链路

```
用户粘贴/修改文本 → debounce 1.2s → POST /api/extract（扩展版，mode='full'）
                                         ↓
                              后端 LLM 同时返回：
                              ① 字段提取（已有）
                              ② catLevel1/2/3 推断
                              ③ matchType: 'exact' | 'fuzzy' | 'none'（前端兜底模糊匹配）
                              ④ suggestedModules
                              ⑤ excludeModules
                                         ↓
                              前端自动填入：
                              ① 表单字段（已有）
                              ② 三级类目下拉框（exact 直接填，fuzzy 填+橙色提示，none 忽略）
                              ③ 模块勾选 = 类目默认 ± 深推调整（不影响 mandatory 和 userTouched 模块）
```

---

## 执行步骤

### Step A：后端 — 扩展 /api/extract

**文件**：`server/index.js`

- 新增 `mode` 参数：`'full'`（默认，字段+类目+模块）| `'fields-only'`（仅字段提取，兼容旧行为）
- Prompt 中传入完整 CATEGORIES 类目树（一级→二级→三级），要求 LLM 从类目树中**精确选择**
- 返回扩展字段：`catLevel1`, `catLevel2`, `catLevel3`, `matchType`, `suggestedModules`, `excludeModules`
- 清理旧 `subCategory` 逻辑，统一迁移到 `catLevel1/2/3`

**验证**：curl POST /api/extract 返回含类目字段

---

### Step B：前端 — 模糊匹配兜底

**文件**：工具函数（放在 `moduleRegistry.ts` 或新建 `utils/categoryMatcher.ts`）

```ts
function fuzzyMatchCategory(llmValue: string, candidates: string[]): { matched: string; type: 'exact' | 'fuzzy' } | null
```

- 先 exact match
- 再 Levenshtein 距离 ≤ 2 的模糊匹配（阈值可调）
- 全都不匹配返回 null

---

### Step C：前端 — LeftPanel Debounce + 自动触发

**文件**：`LeftPanel.tsx`

**C1. Debounce 监听**
- `useEffect` 监听 `pasteText`，1.2s debounce
- 触发条件：非空 && 长度变化 >20% 或新增 >50 字符（小co #6）
- 加载态：类目下拉框旁脉冲条动画（小co #8）

**C2. 收到 LLM 结果后**
- 字段提取 → 已有逻辑
- 类目推断结果做模糊匹配兜底（Step B）
  - exact → 填入类目，绿色小字"AI 已识别类目"
  - fuzzy → 填入类目，橙色小字"AI 推断，需确认"
  - none → 不填类目，灰色小字"未识别到匹配类目，请手动选择"
- 模块深推：
  ```
  base = getDefaultModules(catL1, catL2, catL3)
  final = base + suggestedModules - excludeModules（mandatory 不可移除）
  ```
  - 新模块按 DEFAULT_MODULE_ORDER 插入正确位置（小co #5）
  - 深推只影响 userTouchedModules 中未记录为用户操作过的模块（小co #5）

**C3. 竞态防护**
- `userManuallyChangedCategory` ref：用户手动改类目后置 true（小co #1）
- debounce 回调检测：true → 跳过类目覆盖，模块深推仍执行
- 文本实质性变化时（新商品）→ 重置为 false

**C4. useEffect 冲突防护**
- `autoCategoryRef` ref：AI 填入类目时置 true（小co #2）
- 现有 useEffect（catLevel1/2/3 监听）检测：true → 跳过 moduleOrder 重置
- onChange 完成后重置为 false

**C5. 清空文本**
- pasteText 变为空 → 重置类目为 ''，模块为默认，重置所有 ref 标记（小co #9）

**C6. 清理遗留 subCategory**
- `handleExtractInfo` 中移除 `subCategory` 的 `['dairy','snack',...]` 校验（小co #7）

---

### Step D：构建验证

- `npx vite build` 无报错
- 端到端测试：粘贴商品文本 → 1.2s后类目自动填入 → 模块自动调整
