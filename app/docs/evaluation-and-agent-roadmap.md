## 一、科学评估体系总览

### 1.1 三层评估框架

```
┌──────────────────────────────────────────────────┐
│           L3 业务指标（月度，运营/用户数据）          │
│  单款制作耗时 / 转化率 / 满意度 NPS                  │
├──────────────────────────────────────────────────┤
│         L2 人工评测（周度，抽样 10 款商品）            │
│  风格匹配 / 编辑率 / 双盲打分 / 可用性 SUS             │
├──────────────────────────────────────────────────┤
│       L1 自动化指标（每次生成实时检测）                │
│  合规通过率 / 结构完整度 / 参数准确率 / 幻觉检测         │
└──────────────────────────────────────────────────┘
```

### 1.2 L1 自动化指标（每次生成必测，可机器执行）

| 指标 | 计算方式 | POC 目标 | 最终目标 | 当前状态 |
|------|---------|---------|---------|---------|
| **合规通过率** | post_check 命中数 / 总模块数 | ≤ 2 个/篇 | 0 个/篇 | ⚠️ 后端已实现，前端未展示 |
| **结构完整度** | 成功生成模块数 / 请求模块数 | 100% | 100% | ✅ 已实现 |
| **参数准确率** | 价格/规格/配料与输入一致的比例 | ≥ 90% | ≥ 98% | ⬜ 需开发：正则提取 + 比对 |
| **幻觉检测** | 出现了输入中没有的认证/奖项/功效词数量 | ≤ 2 次/篇 | 0 次 | ⬜ 需开发：NER 实体比对 |

### 1.3 L2 人工评测（周度抽样 10 款，覆盖不同品类）

| 指标 | 方法 | POC 目标 | 最终目标 |
|------|------|---------|---------|
| **风格匹配度** | 5 分制盲测：AI 稿混入人工稿，运营盲评风格贴合度 | ≥ 3.5/5 | ≥ 4.0/5 |
| **人工编辑率** | diff 算法计算运营最终上架版与 AI 初稿的差异字数占比 | ≤ 45% | ≤ 20% |
| **双盲打分** | AI 稿 vs 人工稿，资深运营对打分（不看来源） | AI ≥ 人工×0.80 | AI ≥ 人工×0.95 |
| **可用性 SUS** | 标准系统可用性量表（10 题） | ≥ 65 | ≥ 80 |

### 1.4 L3 业务指标（月度追踪）

| 指标 | 数据来源 | 说明 |
|------|---------|------|
| **单款制作耗时** | 运营工时记录 | AI 辅助 vs 纯人工，目标缩短 60% |
| **转化率对比** | 快团团后台 | AI 文案 vs 人工文案的同款商品笔记转化率 |
| **运营满意度 NPS** | 月度问卷 | "你愿意把这个工具推荐给其他运营吗？"（0-10） |

### 1.5 数据反馈闭环（评测驱动优化）

```
每次生成
  ↓
L1 自动检测 → 标记问题（合规违规/参数错误/结构缺失）
  ↓
每周抽样 10 款
  ↓
L2 人工评测 → 打分 + 记录编辑 diff
  ↓
每月汇总
  ↓
L3 业务数据 → 耗时/转化/NPS
  ↓
分析报告 → 决策：
  ├─ 某模块编辑率高 → 优化该模块 Prompt
  ├─ 某风格评分低 → 细化句式模板 + 禁止用语
  ├─ 某品类幻觉多 → 补录该品类语料
  ├─ 新违规词出现 → 扩充 forbidden_words 规则库
  └─ 运营提出新需求 → 进入需求池排期
```

## 二、阶段一：半自动辅助 Agent

### 2.1 阶段定位

> 纯文本生成工具。运营手动录入商品信息 → AI 生成文案 → 运营审核编辑 → 导出使用。

**Agent 属性：**
- 工具调用：❌ 无
- 自主决策：❌ 无
- 任务规划：❌ 无
- **本质：模块化 Prompt 驱动的生成器，人工配置 + 人工审核**

### 2.2 核心能力（当前已实现 + 即将补齐）

| 能力 | 状态 | 说明 |
|------|------|------|
| 商品信息表单录入 | ✅ | 20+ 字段，对齐 PRD Table 4/5/6 |
| 模块化 Prompt 拼接 | ✅ | 5 必选 + 5 可选，按顺序生成 |
| 双版本差异化 | ✅ | V1 口感侧重 / V2 性价比侧重 |
| 5 种风格切换 | ✅ | 每种风各有句式模板+禁止用语 |
| 流式逐字输出 | ✅ | SSE 逐 token 渲染 |
| 采纳 → 编辑 → 导出 Word | ✅ | 全链路闭环 |
| 合规扫描 | ✅ | pre_banned 阻断 + post_check 标记 |
| AI 对话微调 | ✅ | 流式修改 + 撤销/重做 |
| 文本块自定义 | ✅ | 用户可添加/删除/拖拽自定义模块 |
| **自动评估面板** | ⬜ 短期 | 每次生成后展示 L1 指标结果（合规/结构/参数） |
| **编辑 diff 记录** | ⬜ 短期 | 记录运营每个模块的修改，供后续分析弱点 |
| **品类扩充** | ⬜ 持续 | 按乳制品→休闲零食→生鲜水果→粮油调味→其他顺序，逐品类补录语料 10+ 条，调优 Prompt |

### 2.3 此阶段的评测闭环

```
┌──────────────────────────────────────────────────┐
│  每次生成：L1 自动化指标自动检测                      │
│  ↓                                               │
│  标记问题模块（如"物流售后"参数准确率低）              │
│  ↓                                               │
│  每周：运营试用 10 款不同品类商品                     │
│  ↓                                               │
│  L2 评测：盲测打分 + 记录编辑 diff                   │
│  ↓                                               │
│  分析：哪个模块编辑率最高？哪个风格评分最低？            │
│  ↓                                               │
│  优化：调 Prompt / 补语料 / 扩规则库                  │
│  ↓                                               │
│  下一轮评测 → 对比指标变化 → 验证优化效果              │
└──────────────────────────────────────────────────┘
```

**此阶段的核心优化变量：**
- Prompt 模板（模块指令、风格指令、句式模板）
- 合规规则库（forbidden_words_dev.json）
- 品类语料库（food_rag_corpus）

**评测闭环的节奏：**
- 每周：L1 自动报告 + L2 抽样评测
- 每两周：Prompt 调优一轮
- 每月：L3 业务数据 + 语料库补录一轮

### 2.4 阶段一验收标准

| 验收项 | 标准 |
|--------|------|
| 覆盖品类 | ≥ 3 个一级类目（乳制品、休闲零食、生鲜水果） |
| 人工编辑率 | ≤ 45% |
| 双盲评分 | AI ≥ 人工 × 0.80 |
| 风格匹配 | ≥ 3.5/5 |
| 合规通过率 | post_check 命中 ≤ 1 个/篇 |
| 可用性 SUS | ≥ 65 |

## 三、阶段二：结构化决策 Agent

### 3.1 阶段定位

> 从"手动填表"升级为"上传即生成"。运营上传供应商资料 → 系统自动提取参数填充表单 → 运营选择风格和模块 → 生成图文混排笔记 → 运营审核定稿。

**Agent 属性：**
- 工具调用：✅ 文档解析工具（提取参数）
- 自主决策：❌（风格/模块仍由人工选择）
- 任务规划：❌
- **本质：具备工具调用能力的辅助 Agent，减少人工录入，但核心决策权仍在运营手中**

### 3.2 新增能力

| 能力 | 说明 |
|------|------|
| **文档自动解析** | 上传供应商 Word/PDF/图片 → 自动提取商品名、品类、规格、价格、卖点、配料、产地、保质期等字段 → 自动填入表单。运营仅需核对修正 |
| **图片排版** | 支持在画布中插入图片后，拖拽调整位置、宽度、对齐方式，图片与文字自然混排 |
| **多品类 Prompt 适配** | 每个品类（乳制品/零食/水果/粮油）有独立的模块 Prompt 微调版本，避免「乳制品 Prompt 写零食」的生硬感 |
| **评测数据看板** | 运营和管理者可见：本周编辑率趋势、各模块编辑率排行、风格评分排行、合规命中分布 |

### 3.3 此阶段的评测闭环

```
┌──────────────────────────────────────────────────┐
│  文档解析：上传 → 自动提取 → 对比人工修正量            │
│  ↓（衡量解析准确率）                                │
│  生成：L1 自动检测 + 记录编辑 diff                    │
│  ↓                                               │
│  每周 L2 抽样评测（覆盖所有已支持品类）                 │
│  ↓                                               │
│  月度 L3 业务数据（耗时/转化/NPS）                     │
│  ↓                                               │
│  分析报告：                                        │
│  - 文档解析准确率（分品类统计）                       │
│  - 各模块编辑率趋势（是否持续下降？）                  │
│  - 各风格评分趋势（是否稳定提升？）                    │
│  - 不同品类的生成质量差异（哪个品类最差？优先优化）       │
│  ↓                                               │
│  决策：调 Prompt / 扩充品类语料 / 优化解析模型          │
└──────────────────────────────────────────────────┘
```

**新增评测指标：**

| 指标 | 计算方式 | 目标值 |
|------|---------|--------|
| **文档解析准确率** | 正确提取的字段数 / 总字段数 | ≥ 85% |
| **图片排版满意度** | 运营 5 分制评价图片位置的合理性 | ≥ 3.5/5 |
| **品类覆盖度** | 已适配 Prompt 的品类数 | ≥ 5 个 |

### 3.4 阶段二验收标准

| 验收项 | 标准 |
|--------|------|
| 文档解析准确率 | ≥ 85% |
| 人工编辑率 | ≤ 30% |
| 双盲评分 | AI ≥ 人工 × 0.90 |
| 覆盖品类 | ≥ 5 个 |
| 图片排版 | 支持拖拽调整位置和尺寸 |
| 评测看板 | 周度自动生成报告 |

## 四、阶段三：全流程执行 Agent

### 4.1 阶段定位

> AI 全面接管决策。运营仅需上传资料 + 最终审核发布。AI 自动完成：参数提取 → 风格选择 → 模块组合 → 生成 → 合规校验 → 排版优化 → 待发布。

**Agent 属性：**
- 工具调用：✅ 文档解析、平台 API、图片处理
- 自主决策：✅ 风格推荐、模块推荐、排版决策
- 任务规划：✅ 多步骤串行闭环执行
- **本质：接近完整的自主 Agent，人工仅做最终审核和确认发布**

### 4.2 为什么此时才能让 AI 做决策

| 条件 | 说明 |
|------|------|
| **语料充足** | 5 个品类 × 5 种风格 × 每个模块 ≥ 10 条语料 = 250+ 条。AI 有足够参照，不再依赖 Prompt 硬约束 |
| **评测数据积累** | 阶段一、二积累了数月的编辑 diff + 评分数据，AI 知道「什么风格适合什么品类」「什么模块组合转化好」 |
| **规则库成熟** | 合规规则经过多轮扩充，覆盖面足够 |
| **生成稳定性验证** | 参数准确率、幻觉率已在阶段二达到目标 |

### 4.3 新增能力

| 能力 | 说明 |
|------|------|
| **自动风格决策** | AI 根据商品价格带、品类、目标人群、活动类型，自动推荐最佳风格（运营可覆盖） |
| **自动模块组合** | AI 根据商品特征（是否促销、是否新品、是否高端），自动决定启用哪些可选模块 |
| **RAG 精准注入** | 语料库 250+ 条 → 按品类+风格+模块精准检索 Top3 → 注入 Few-shot → 生成质量稳定 |
| **一键发布** | 对接快团团 API，审核通过后直接发布上架 |
| **A/B 测试引擎** | 同一款商品自动生成 2 个版本发布，追踪转化数据，反馈到风格/模块决策模型 |
| **个性化偏好学习** | 学习每个用户的历史修改习惯（喜欢什么风格、经常改什么模块），自动调整生成偏好 |

### 4.4 此阶段的评测闭环

```
┌──────────────────────────────────────────────────┐
│  全自动流程：上传 → AI 决策 → 生成 → 人工审核         │
│  ↓                                               │
│  L1 自动检测 + L2 人工评测 + L3 业务数据              │
│  ↓                                               │
│  新增：A/B 测试数据（哪个版本转化高？）                │
│  ↓                                               │
│  新增：AI 决策准确率（风格推荐被采纳率？）              │
│  ↓                                               │
│  自优化循环：                                      │
│  - 转化高的风格/模块组合 → 增加权重                   │
│  - 被频繁修改的模块 → 自动调整 Prompt                  │
│  - 新违规模式 → 自动建议加入规则库                     │
│  - 优秀生成结果 → 一键收录到语料库                     │
│  ↓                                               │
│  月度评估报告 → 验证自优化效果                        │
└──────────────────────────────────────────────────┘
```

**新增评测指标：**

| 指标 | 计算方式 | 目标值 |
|------|---------|--------|
| **风格推荐采纳率** | 运营接受的风格推荐次数 / 总推荐次数 | ≥ 75% |
| **模块推荐采纳率** | 运营接受的模块组合 / 总推荐次数 | ≥ 80% |
| **AI 决策满意度** | 运营对 AI 自动选择的 5 分制评价 | ≥ 4.0/5 |
| **A/B 测试胜出率** | AI 版转化 ≥ 人工版的比例 | ≥ 60% |

### 4.5 阶段三验收标准

| 验收项 | 标准 |
|--------|------|
| 人工编辑率 | ≤ 15% |
| 双盲评分 | AI ≥ 人工 × 0.95 |
| 全流程耗时 | ≤ 3 分钟/款（含审核） |
| 风格推荐采纳率 | ≥ 75% |
| A/B 转化 | AI 版不差于人工版 |
| 语料库规模 | ≥ 250 条 |
