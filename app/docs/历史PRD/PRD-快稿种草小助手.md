## 一、产品概述

快稿种草小助手是一个 AI 驱动的商品笔记内容生成工具。运营输入商品信息 + 上传图片，系统自动生成图文混排的种草笔记，支持三版并行对比、逐模块采纳编辑、一键导出。

**产品定位**：从"AI帮你写"到"AI越来越像你写"——系统通过学习运营的采纳、编辑和导出行为，逐步适配个人风格。

**发布方式**：独立网址（独立平台），后续对接一键发布到快船、群接龙等工作台。

## 四、核心功能

### 4.1 商品信息录入

#### 字段分类（2026-07-20 运营会议确认）

**必填字段：**

| 字段 | 说明 | 影响模块 |
|------|------|------|
| 商品名称 | — | 全文 |
| 规格净含量 | 支持多规格（一瓶/两瓶/不同克重） | price |
| 建议售价 | 支持多价格对应 | price |
| 核心配料/原料 | — | ingredient, trust |
| 保质期 | — | tips, faq |

**待开发：多规格/多价格动态表单。** 当前为单一输入框，需重构为支持量词、多规格、多售价对应的动态适配区。

**非必填字段（供应商提供，不强制）：**

| 字段 | 说明 |
|------|------|
| 产地 | 有则填 |
| 发货地 | 供应商提供 |
| 快递公司 | 供应商提供 |
| 发货时效 | 供应商提供 |
| 补邮费/不发货地区 | 供应商提供 |

**物流售后处理方式（会议确认）：** 不作为卖点融入正文，而是以统一段落形式直接贴在笔记末尾。售后规则不默认勾选。

#### AI 提炼卖点（待开发）

当供应商给的图文素材极少时，系统从图片 OCR 文字或文档段落中自动识别信息，提炼出核心卖点供运营勾选确认。替代当前的"卖点润色"单向交互。

### 4.2 图片处理

```
上传图片 → 压缩（客户端 canvas, 512px宽）→ 图片指纹去重
  → doubao vision API 分析（type/desc/layout_role/imageContentSummary/imageOcrText/suggestedModule）
  → 运营在确认弹窗中确认/修正
  → 参与生成
```

**图片压缩规范**：运营提交的 Word 文档中图片需预先压缩（当前几百MB导致卡顿）。需输出压缩指南或开发预处理工具。

### 4.3 模块选择

左面板展示所有可用模块（按类目动态过滤），运营勾选本次笔记需要的模块。

- **必选模块**：灰色不可取消（hook/price/cta）
- **推荐模块**：默认勾选，可取消
- **可选模块**：默认不勾选，doubao 识别到相关图片时自动勾选

**左面板不提供排序功能。** 模块排列顺序由生成引擎各自决定（见 §六），运营可在中栏拖拽调整。

### 4.4 一键生成

三版并行生成，各自使用独立的模块顺序和风格 prompt。详见 §六。

### 4.5 中栏编辑

- contentEditable 编辑器，支持直接打字、粘贴图片、拖拽图片
- 模块卡片可拖拽排序（唯一排序来源）
- 快捷按钮：扩充、口语化、排版、加/去 emoji、强化卖点等
- 对话框自由输入：非 chip 触发的自然语言改写指令

### 4.6 比价功能

**首发版**：人工截图对比。运营上传竞品价格截图，圈出价格并标注对比信息。系统以文字列表展示。

**长期**：预研自动抓取价格（反爬是核心难点），规划自动生成比价图。

### 4.7 导出与发布

- **导出**：下载 .doc 文件 + 自动存入预审语料库（corpus-review/）+ 触发学习数据采集
- **发布**（待开发）：一键同步到快船、群接龙等工作台

### 4.8 AI 图片生成

**V2.0+ 规划，首发版不集成。**

过渡期方案：人工拍摄商品 360 度照片 → AI 背景移除（抠图） → 生活场景美化合成。

## 五、类目与模块体系

### 5.1 三级类目结构

15 个一级类目，500+ 三级类目。采用 `level1::level2::level3` 编码。

### 5.2 模块注册表

**通用模块（10个，所有类目共享）：**
hook, price, trust, brand, scene, aftercare, tips, cta, feedback, faq

**可选通用模块（按类目配置）：**
comparison（全网比价）

**美食酒水专属模块（3个）：**
taste（口感体验）, ingredient（成分科普）, origin（原料溯源）

**美妆洗护专属模块（Phase 3）：**
beauty_effect, beauty_ingredient, usage

### 5.3 模块配置规则

三层查找链：`三级精确 → 二级覆盖 → 一级默认 → __default__`

详见《类目驱动模块体系设计方案》（后续需更新以反映本文档变更）。

### 5.4 模块默认顺序（兜底值）

**美食酒水**（2026-07-20 运营会议确认——价格靠前第二顺位）：

```
hook → price → taste → origin → ingredient → trust → brand → scene → aftercare → tips → feedback → comparison → faq → cta
```

> ⚠️ 此顺序与当前代码 `moduleRegistry.ts` 中的 DEFAULT_MODULE_ORDER['美食酒水'] 不同。当前代码 price 在第5位，需更新为第2位。

**美妆洗护：**

```
hook → beauty_effect → beauty_ingredient → usage → price → trust → brand → scene → aftercare → tips → feedback → comparison → faq → cta
```

**其他类目（兜底）：**

```
hook → price → trust → brand → scene → aftercare → tips → feedback → comparison → faq → cta
```

### 5.5 物流售后特殊处理

aftercare 模块在所有类目中的处理方式：
- 不作为卖点融入正文
- 以统一段落形式贴在笔记末尾
- 由系统自动拼接（发货地+快递+时效+补邮费+不发货地区+售后规则）
- 默认不勾选

## 六、三版生成引擎

### 6.1 三版定位

| 版本 | 标签 | 风格来源 | 模块顺序来源 |
|------|------|------|------|
| V1 | 默认风格 | 固定位置，个人进化（初始=小红书，逐步融合 L2 画像参数） | 用户学习（从该品类历史中栏定稿学出） |
| V2 | Elo #1 风格 | Elo 排名第一的纯风格（冷启动：xiaohongshu） | 语料库参考（同风格高分语料） |
| V3 | Elo #2 风格 | Elo 排名第二的纯风格（冷启动：fun） | 语料库参考（同风格高分语料） |

6种纯风格：xiaohongshu / girlfriend / fun / minimalist / premium / senior

### 6.2 生成流程

```
一键生成
  → 确定 V2/V3 风格（Elo排名 or DEFAULT_INPUT.versionStyles 冷启动兜底）
  → 取 selectedModules
  → 并行取序：
      V1 = userLearnedOrder(catLevel1, catLevel2, catLevel3)
      V2 = await corpusReferenceOrder(catLevel1, catLevel2, catLevel3, v2Style)
      V3 = await corpusReferenceOrder(catLevel1, catLevel2, catLevel3, v3Style)
  → 各版过滤 selectedModules
  → 并行 streamGenerate × 3
```

### 6.3 用户学习顺序

三层查找：`三级精确 → 二级聚合 → 一级聚合 → DEFAULT_MODULE_ORDER`

- 聚合按使用频次加权
- 某三级类目 ≥ 10 次导出后独立

### 6.4 语料参考顺序

```
GET /api/corpus/search → 动态评分排序 → Top-3 加权随机 → 取 moduleOrder → 过滤+补充
无结果 → 兜底 DEFAULT_MODULE_ORDER
```

语料评分公式：`质量分 × 验证分 × 时效分`，系统自动计算。详见《语料动态评分系统设计方案》。

## 七、学习系统

### 7.1 触发时机

所有学习在**导出时**触发（异步，不阻塞）。

### 7.2 五个学习维度

| 维度 | 内容 | 存储 | 阶段 |
|------|------|------|:--:|
| 模块顺序 | 中栏定稿的最终模块排列 | localStorage 按类目 | Phase 1 |
| 图片排版 | 文字-图片顺序变化、图片数量增减 | localStorage | Phase 1 |
| 风格偏好 | 采纳/点踩 → Elo 排名 | localStorage（数据飞轮 L1） | 后续 |
| 文风句式 | LLM 分析编辑 diff → 偏好参数 | localStorage（数据飞轮 L2） | 后续 |
| 编辑距离 | 采纳快照 vs 定稿的编辑率 | localStorage + corpus JSON | Phase 1 |

### 7.3 采纳快照

采纳时记录 AI 原文 hash + 图片 ID（不存 base64 和全文），导出时对比定稿计算编辑率。

## 八、语料体系

### 8.1 语料结构（v2.1 schema 扩展版）

```json
{
  "version": "2.1",
  "schema": "corpus-图文绑定-v2",
  "productName": "...",
  "styleTag": "xiaohongshu",
  "editRate": 0.15,
  "moduleOrderOriginal": true,
  "referenceCount": 16,
  "moduleOrder": ["hook","price","taste",...],
  "imageLayout": {"hook":"image_last","taste":"interleave"},
  "category": {"level1":"美食酒水","level2":"酒水饮料","level3":"乳制品"},
  "convertedAt": "2026-07-18T00:00:00.000Z",
  "modules": [{
    "moduleKey": "hook",
    "order": 1,
    "segments": [{"text":"...","images":[{"sourceImageId":"abc123","imgId":1}]}]
  }]
}
```

### 8.2 语料生命周期

```
运营导出 → 自动写入 corpus-review/ → 运营审核 → 移入 corpus/
  → import_corpus.cjs 入库 → RAG 索引更新
```

### 8.3 语料评分

系统自动动态评分，不用人打分。公式见《语料动态评分系统设计方案》。

### 8.4 冷启动

- **第一批语料**：运营从美食酒水各二级类目收集 3-5 篇优质笔记（Word 格式，需控制图片大小）
- **风格覆盖**：当前仅有 xiaohongshu 语料，其他风格逐步积累
- **无匹配语料** → 兜底 DEFAULT_MODULE_ORDER

## 九、技术架构

### 9.1 技术栈

| 层 | 方案 |
|------|------|
| 前端 | React + TypeScript + Vite + Tailwind CSS |
| 后端 | Node.js Express |
| 文本 AI | DeepSeek API |
| 图片 AI | doubao-seed-2.0-lite（vision API）|
| 图片指纹 | 客户端 canvas 采样 hash + 服务端 Hamming distance 去重 |
| 存储 | 本地文件系统（JSON + JSONL） |

### 9.2 API 清单

| 端点 | 说明 | 状态 |
|------|------|:--:|
| `POST /api/generate/stream` | 流式生成文案 | ✅ |
| `POST /api/images/classify` | doubao 图片分类 | ✅ |
| `POST /api/images/match-corpus` | 图片指纹匹配 | ✅ |
| `GET /api/corpus/image-map` | 图片→模块映射 | ✅ |
| `POST /api/corpus/save-to-review` | 导出存预审库 | ✅ |
| `POST /api/extract` | 文件解析提取字段 | ✅ |
| `POST /api/chat/stream` | 对话框改写 | ✅ |
| `GET /api/corpus/search` | 语料搜索（动态评分排序） | 🔜 |
| `POST /api/corpus/reference` | 递增语料引用计数 | 🔜 |
| `POST /api/signals/report` | 学习信号上报 | 🔜 |

### 9.3 部署

- 前端：Cloudflare Pages（免费）
- 后端：Render / Railway（免费层）
